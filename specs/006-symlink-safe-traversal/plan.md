# Implementation Plan: Symlink-Following, Cycle-Safe Folder Traversal

**Branch**: `006-symlink-safe-traversal` | **Date**: 2026-06-22 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/006-symlink-safe-traversal/spec.md`

## Summary

Add symlink-following and cycle-safe directory traversal to the filesystem
infrastructure adapter. `os.walk` is changed to `followlinks=True` in both traversal
passes. A new `VisitedDirectoryTracker` (keyed on `(st_ino, st_dev)`) guards against
cycles and duplicate directory visits, with a configurable hard cap that aborts the scan
cleanly if exceeded. The tracker is shared across the hint-promotion pass and the
candidate-file discovery pass within a single scan. No new runtime dependencies.

### Task 3 — `VisitedDirectoryTracker` and `DirectoryLimitExceededError`

**File**: `adapters/filesystem.py` (top of file, before existing functions)

```python
class DirectoryLimitExceededError(RuntimeError):
    def __init__(self, limit: int, path: Path) -> None:
        self.limit = limit
        self.path = path
        super().__init__(
            f"Directory visit limit ({limit}) reached at: {path}. "
            "Scan is incomplete."
        )


class VisitedDirectoryTracker:
    def __init__(self, limit: int) -> None:
        self._visited: set[tuple[int, int]] = set()
        self._limit = limit

    def try_visit(self, path: Path) -> bool:
        """Stat path, record as visited, return True if new.

        Returns False if already visited.
        Raises OSError if path cannot be stat'd (broken/inaccessible link).
        Raises DirectoryLimitExceededError if the cap is reached.
        """
        stat = path.stat()
        key = (stat.st_ino, stat.st_dev)
        if key in self._visited:
            return False
        if len(self._visited) >= self._limit:
            raise DirectoryLimitExceededError(self._limit, path)
        self._visited.add(key)
        return True

    @property
    def visited_count(self) -> int:
        return len(self._visited)
```

**Test impact**: New `tests/unit/test_visited_directory_tracker.py` (see Task 6).

### Task 4 — Shared `_prune_with_visit_check` Helper

**File**: `adapters/filesystem.py`

Extract the repeated "stat subdirectory, prune if visited, log, handle errors" pattern
into a private helper to satisfy DRY:

```python
_logger = logging.getLogger(__name__)

def _prune_with_visit_check(
    dirnames: list[str],
    current_path: Path,
    ignore_set: frozenset[str],
    tracker: VisitedDirectoryTracker,
    issues: list[DiscoveryIssue] | None = None,
) -> None:
    """Mutate dirnames in-place: remove ignored, already-visited, or inaccessible dirs.

    Raises DirectoryLimitExceededError if the tracker cap is hit.
    """
    safe: list[str] = []
    for name in dirnames:
        if name in ignore_set:
            continue
        sub = current_path / name
        try:
            if tracker.try_visit(sub):
                safe.append(name)
            else:
                _logger.debug("skipping already-visited directory: %s", sub)
        except DirectoryLimitExceededError:
            raise
        except OSError as exc:
            _logger.debug("skipping inaccessible link: %s (%s)", sub, exc)
            if issues is not None:
                issues.append(DiscoveryIssue(location=sub, error_type=type(exc).__name__))
    dirnames[:] = safe
```

This helper replaces the inline `dirnames[:] = [d for d in dirnames if d not in ignore_set]`
calls in both walk sites.

### Task 9 — Unit Tests: `VisitedDirectoryTracker`

**File**: `tests/unit/test_visited_directory_tracker.py`

Test cases (all use `tmp_path` for real directories):

1. New real directory → `try_visit` returns `True`, `visited_count` increments.
2. Same directory via same path again → returns `False`.
3. Same directory via a different path (symlink to it) → returns `False` (same inode).
4. Two distinct directories → both return `True`, `visited_count = 2`.
5. Limit = 1, try to visit a second directory → `DirectoryLimitExceededError` raised,
   `.limit` and `.path` attributes correct.
6. Broken/non-existent path → `OSError` propagated (not swallowed).
7. `visited_count` reflects number of unique entries.
